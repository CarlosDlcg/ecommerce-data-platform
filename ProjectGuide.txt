Project Requirements
1. Project Goal
Build a production-ready automated data pipeline using:
•	Python → development
•	Apache Spark → distributed processing
•	dbt → transformations & SCD2
•	Apache Airflow → orchestration
The pipeline must:
•	Integrate multiple sources
•	Track historical changes (SCD2)
•	Run automatically every day
•	Include monitoring and recovery mechanisms
2. Functional Requirements
Multi-Source Data Ingestion
Pipeline must extract data from at least three source types:
•	Database tables
•	REST APIs
•	CSV files
Data Quality Validation
Validate:
•	Null values
•	Data types
•	Business rules
Goal:
•	Ensure trustworthy downstream analytics.
Incremental Processing
Process only:
•	New records
•	Updated records
Benefits:
•	Faster execution
•	Lower resource consumption
SCD2 Historical Tracking
Maintain:
•	Full history of changes
•	Effective dates
•	Current record indicators
For key business entities (especially customers).
Modular Transformations
Organize logic into:
•	Reusable functions
•	Independent components
•	Testable modules
Error Handling & Recovery
Must support:
•	Retries
•	Logging
•	Alerting
•	Recovery after failures
Automated Scheduling
Pipeline executes:
•	Daily
•	Through Airflow DAGs
Data Lineage
Document:
•	Source systems
•	Transformations
•	Destination tables
Goal:
•	Trace data throughout the pipeline.

Only process:
